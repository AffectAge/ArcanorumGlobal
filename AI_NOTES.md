# Изменения — Этап 1: WS дельты вместо полного worldBase

## Что сделано
- Переведен протокол обновления мира с `WORLD_PATCH`/`WORLD_BASE_SYNC` на `WORLD_DELTA` с версией состояния (`worldStateVersion`).
- Добавлен расчет дельт на сервере:
  - снапшот мира до мутаций;
  - вычисление изменений по веткам `resourcesByCountry`, `provinceOwner`, `provinceNameById`, `colonyProgressByProvince`, `provinceColonizationByProvince`;
  - tombstone-удаления через `null` для удаляемых ключей.
- Обновлен `AUTH_OK`: теперь отдает полный снапшот мира + `worldStateVersion`.
- Убраны рассылки полного `worldBase` по событиям, где раньше использовался `broadcastWorldBaseSync`.
- На клиенте добавлен merge-применитель дельт в Zustand (`applyWorldDelta`) вместо полной замены `worldBase` на каждом апдейте.
- На клиенте добавлена проверка последовательности `worldStateVersion`; при рассинхроне выполняется жесткое переподключение (reload), без legacy fallback.
- Обновлены общие типы в `packages/shared` под новый протокол.

## Почему так
- Полный `worldBase` в каждом WS-сообщении плохо масштабируется по сети и сериализации.
- Дельты уменьшают payload и снижают нагрузку на сервер/клиент.
- Версионирование состояния исключает "тихий" рассинхрон при пропуске сообщений.
- Полный снимок остается только в `AUTH_OK`, что соответствует non-legacy подходу без отдельного sync fallback-события.

## Риски / ограничения
- Дельты зависят от корректности diff/merge; ошибки в одной ветке могут дать рассинхрон.
- При рассинхроне сейчас применяется жесткий reload (ожидаемое поведение без fallback, но UX резче).
- Для глубоких структур используется сравнение через `JSON.stringify` в части веток; на очень больших объемах это может стать дополнительной CPU-нагрузкой.
- `worldStateVersion` в текущем этапе не персистится в БД (после рестарта это не критично, т.к. клиенты получают новый полный `AUTH_OK`).

## Как проверить
1) Сборка/типизация:
- `npm run typecheck -ws`

2) Запуск:
- `npm run dev`

3) Ручные сценарии:
- Войти двумя клиентами, выполнить колонизацию/отмену/переименование провинции и убедиться, что мир обновляется без `WORLD_BASE_SYNC` событий.
- Завершить ход и проверить, что приходит `WORLD_DELTA` с `rejectedOrders`, а ресурсы/владение/прогресс корректно применяются.
- Через админку изменить настройки, влияющие на стоимость колонизации, и проверить, что клиенты получают только дельту.
- Смоделировать рассинхрон (временно подменить `worldStateVersion` в клиенте) и убедиться, что происходит принудительный reload.
