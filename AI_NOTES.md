# Изменения — Этап: добыча ресурсов провинции зданиями

## Что сделано
- Добавлена механика добычи из провинциальных залежей в серверном экономическом тике (`resolveBuildingsTurn`):
  - здание может добывать ресурс из `provinceResourceDepositsByProvince`,
  - добытый объем зависит от `productivity` и базового `extractionAmountPerTurn`,
  - при `extractionRequiresDeposit=true` залежь уменьшается на фактически добытый объем.
- Добыча интегрирована с текущей экономикой зданий:
  - ресурс кладется в `instance.warehouseByGoodId`,
  - учитывается в `lastProductionByGoodId`,
  - добавлен отдельный след `lastExtractionByGoodId` для диагностики.
- Обновлены типы и сериализация/нормализация инстансов зданий:
  - `packages/shared/src/index.ts`: `BuildingInstance.lastExtractionByGoodId`.
  - `apps/server/src/index.ts`: normalize/compare/reset для нового поля.
- Расширены настройки контент-записей зданий (сервер + клиент API):
  - `extractionGoodId`,
  - `extractionAmountPerTurn`,
  - `extractionRequiresDeposit`.
- В `ContentPanel` (админка контента) добавлен UI-блок настройки добычи для зданий.
- README обновлен описанием новой механики.

## Почему так
- Требование: дать зданиям возможность использовать залежи ресурсов провинции как источник добычи.
- Реализация через контент-поля здания минимально инвазивна: не ломает существующую торговлю/склады и использует уже существующие данные по залежам.

## Риски / ограничения
- Добыча сейчас работает на уровне инстанса здания и прямого доступа к залежам провинции; нет отдельной системы лицензий/квот.
- `infraCoverage` в расчете здания пока фиксирован как `1` (как и было), поэтому ограничение добычи идет через продуктивность и наличие залежей, а не отдельный cap инфраструктуры добычи.
- Баланс добычи зависит от ручной настройки `extractionAmountPerTurn` в контенте зданий.

## Как проверить
1) Типизация:
- Выполнить `npm run typecheck -ws`.
- Ожидаемо: ошибок нет.

2) Настройка здания:
- В `ContentPanel` открыть категорию `Здания` -> `Экономика и производство`.
- Задать:
  - `Добываемый товар` (например товар с discoverable-настройками),
  - `Объем/ход > 0`,
  - `Требует залежь = Да`.
- Сохранить.

3) Игровой сценарий:
- Убедиться, что в провинции есть залежь по выбранному товару.
- Завершить ход.
- Проверить, что:
  - в инстансе здания вырос `warehouseByGoodId[goodId]`,
  - в провинции уменьшился `provinceResourceDepositsByProvince[provinceId][goodId].amount`,
  - в телеметрии здания есть `lastExtractionByGoodId`.

4) Проверка совместимости:
- Убедиться, что остальная экономика (покупка входов/продажа/зарплаты/неактивность) работает как раньше.
